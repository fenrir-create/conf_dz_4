Эта программа представляет собой простой ассемблер, который переводит текстовые команды из исходного файла в формат машинного кода, записывая их в бинарный файл. Кроме того, она создает лог-файл в формате CSV, содержащий подробное текстовое описание обработанных инструкций. Основная цель программы — преобразование инструкций высокого уровня в более низкоуровневое представление, пригодное для исполнения машиной.

### Основные функции программы:
1. **Чтение и интерпретация команд**:
   - Программа считывает текстовый файл, содержащий команды и их аргументы, разделённые пробелами. Каждая строка представляет одну инструкцию.
   - Первая часть строки (`opcode`) определяет тип команды, а последующие значения — её аргументы.

2. **Обработка различных инструкций**:
   - Для каждой команды предусмотрен свой алгоритм обработки, который зависит от её кода (`opcode`) и структуры:
     - **LOAD_CONST (код 26)**: Загрузка константы в регистр. Требует три аргумента.
     - **READ_MEM (код 14)**: Чтение из памяти. Требует два аргумента.
     - **WRITE_MEM (код 7)**: Запись в память. Требует три аргумента.
     - **UNARY_ABS (код 6)**: Вычисление абсолютного значения. Требует два аргумента.
   - Если команда неизвестна или имеет неправильное количество аргументов, программа выдаёт ошибку.

3. **Преобразование команд в машинный код**:
   - Каждая команда преобразуется в 32-битное число с помощью побитовых операций:
     - Код команды и аргументы упаковываются в конкретные позиции внутри 32-битного числа.
     - Например, `opcode` занимает старшие биты, а аргументы записываются в оставшихся частях.

4. **Создание бинарного файла**:
   - После обработки каждая инструкция записывается в бинарный файл в формате Big-Endian, что позволяет программам или устройствам интерпретировать машинный код.

5. **Создание лог-файла**:
   - Лог-файл в формате CSV содержит описание всех обработанных команд, включая их название и аргументы. Это полезно для отладки и анализа, так как позволяет в человекочитаемом виде увидеть, какие команды и с какими параметрами были обработаны.

### Обработка ошибок:
Программа проверяет:
- Правильность количества аргументов для каждой команды.
- Наличие в списке инструкций команды с переданным `opcode`.
Если возникает ошибка, программа сообщает о ней, чтобы пользователь мог исправить исходный файл.

### Применение:
Такой ассемблер может быть полезен для обучения, создания небольших виртуальных машин или встроенных систем, где команды необходимо переводить в машинный код. Он также подходит для моделирования поведения процессора и тестирования программ перед запуском на реальном оборудовании.
